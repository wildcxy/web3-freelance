# Web3 接单样品站 PRD

## 1. 项目概述

Web3 接单样品站是一个面向 Web3 兼职投递的个人作品项目。项目通过一个 NFT Mint 演示台展示交付能力，包括 Web3 前端页面、钱包连接流程、NFT mint 状态反馈、合约样例、项目分析和投递话术。

当前版本定位为静态演示 MVP，不连接真实钱包、不发起真实链上交易，适合用于作品集、兼职投递、客户沟通和后续真实 dApp 改造。

## 2. 项目目标

- 让求职者或兼职接单者拥有一个可演示的 Web3 样品站。
- 向项目方证明基本 Web3 产品交付能力。
- 同时覆盖内容、运营、前端和简单合约沟通场景。
- 降低初学者直接接真实合约任务的风险。

## 3. 目标用户

### 3.1 主要用户

- 想找 Web3 兼职的新手或转行者
- 会使用 AI 辅助完成交付的人
- 想接中文内容、社区运营、教程、前端 demo 的个人接单者

### 3.2 次要用户

- 招募 Web3 兼职的项目方
- 需要快速验证 NFT mint 流程的早期项目
- 想了解候选人交付能力的社区负责人

## 4. 使用场景

### 4.1 投递兼职

用户将项目部署到 GitHub Pages、Vercel 或 Netlify，并把链接发给项目方，作为作品集证明。

### 4.2 客户沟通

用户向客户说明自己可以交付哪些 Web3 小任务，例如钱包连接、NFT mint 页面、中文教程、项目分析文档等。

### 4.3 后续升级

用户可以把静态模拟版升级为真实测试网版本，接入 MetaMask、wagmi、viem 和测试网 ERC-721 合约。

## 5. 功能需求

### 5.1 首页展示

- 展示项目名称和定位。
- 展示 Web3 能力标签。
- 展示 NFT 凭证视觉资产。
- 展示当前网络、Supply、价格和 mint 限制。

### 5.2 钱包连接模拟

- 用户点击“连接演示钱包”。
- 页面生成一个模拟钱包地址。
- 状态从“未连接”变为“已连接”。
- Mint 按钮变为可点击。

### 5.3 NFT Mint 模拟

- 用户点击“Mint NFT”。
- Supply 数量增加。
- 页面生成模拟交易哈希。
- Mint 按钮变为“已 Mint”并禁用。
- 页面展示交易状态。

### 5.4 兼职投递话术生成

- 用户输入称呼。
- 用户输入作品链接。
- 用户选择目标任务类型。
- 页面生成一段可复制的中文投递话术。

### 5.5 配套文档

- README：说明项目定位和使用方式。
- PRD：说明产品目标、用户、功能和验收标准。
- 需求架构文档：说明模块、数据流、技术架构和升级路线。
- 项目分析：展示 Web3 项目研究能力。
- 投递模板：提供中英文 outreach 文案。

## 6. 非功能需求

- 页面可以直接通过浏览器打开。
- 不依赖后端服务。
- 不依赖真实钱包。
- 适配桌面和移动端。
- 文案要适合 Web3 兼职投递场景。
- 不涉及真实资金、私钥、助记词或主网交易。

## 7. 验收标准

- 页面可以正常加载。
- NFT 图片可以正常显示。
- 点击连接钱包后，状态、地址和按钮状态正确变化。
- 点击 Mint 后，Supply、交易哈希和按钮状态正确变化。
- 填写投递表单后，可以生成新的投递话术。
- 所有项目文档存在且能解释项目价值。
- 项目目录结构清晰，适合上传 GitHub。

## 8. 后续版本规划

### V1.1 真实钱包连接

- 接入 wagmi 和 viem。
- 支持 MetaMask 或 WalletConnect。
- 读取用户真实钱包地址。

### V1.2 测试网合约部署

- 使用 Foundry 部署 ERC-721 合约到 Base Sepolia。
- 前端调用真实合约 mint。
- 显示真实交易哈希。

### V1.3 作品集增强

- 增加项目案例列表。
- 增加服务报价页。
- 增加中英文切换。
- 增加客户需求表单。

### V1.4 Web3 兼职工具化

- 增加项目分析模板生成。
- 增加 Discord 社区运营周报模板。
- 增加空投/测试网交互教程模板。
