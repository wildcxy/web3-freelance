# Proof Credential NFT Mint Demo 案例说明

## 背景

这个样例面向早期 Web3 项目、社区活动和测试网教程场景。项目方通常需要一个轻量 Demo，用来说明用户如何连接钱包、领取凭证 NFT，并在页面上看到清晰的交易状态反馈。

当前版本不连接真实钱包，不发起真实链上交易，定位是作品集和客户沟通原型。

## 目标

- 展示一个完整的 NFT Mint 用户流程。
- 展示基础 Web3 前端交互能力。
- 展示 ERC-721 合约沟通能力。
- 展示产品文档、项目分析和投递材料整理能力。
- 降低新手直接处理真实资产和主网交易的风险。

## 交付范围

- 响应式 Mint 演示页面。
- 演示钱包连接状态。
- 演示 Mint 后的 Supply 和交易哈希变化。
- ERC-721 合约样例。
- README、PRD、架构文档、项目分析、投递模板和上线检查清单。
- 面向兼职投递的服务说明和联系入口。

## 非交付范围

- 不处理真实用户资产。
- 不请求私钥、助记词或签名授权。
- 不部署主网合约。
- 不承诺合约审计能力。
- 不提供金融收益、空投收益或投资判断。

## 用户流程

1. 用户打开 Demo 页面。
2. 用户查看 NFT 凭证视觉和 Mint 信息。
3. 用户点击“连接演示钱包”。
4. 页面生成模拟钱包地址并更新连接状态。
5. 用户点击 “Mint NFT”。
6. 页面更新 Supply、交易哈希和按钮状态。
7. 用户可生成投递话术并复制。

## 风险控制

- 页面明确标注静态演示边界。
- 合约样例限制每个地址 mint 1 个。
- 合约样例设置最大供应量。
- 当前 Demo 不包含 paid mint 和提现逻辑。
- 后续如升级真实链上版本，需要增加合约测试、部署脚本和交易失败处理。

## 可升级方向

- 使用 wagmi / viem 接入真实钱包。
- 使用 Base Sepolia 测试网。
- 使用 Foundry 增加合约测试和部署脚本。
- 使用 IPFS / Arweave 保存 metadata。
- 增加白名单签名 mint。
- 增加中英文切换和更多作品案例。
