# Web3 兼职投递行动计划

目标不是等别人主动来找，而是用“小任务试做 + 作品链接 + 明确交付物”去打开第一单。

## 优先投递方向

先投这些更容易拿到反馈的任务：

- 中文内容、本地化、教程
- 测试网交互文档
- 项目分析和中文介绍
- Discord / Telegram 社区内容整理
- 钱包连接页、NFT Mint 静态 Demo
- DevRel 辅助材料，比如 FAQ、用户引导、活动说明

暂时不要主攻：

- Solidity 合约开发
- 合约审计
- 主网 paid mint
- 资金相关任务
- 任何需要私钥、助记词、押金或代操作账户的任务

## 每日动作

每天用 60-90 分钟做下面几件事：

1. 找 5 个新项目或任务。
2. 给 3 个项目发定制私信。
3. 对 1 个项目做一个小型试做材料。
4. 记录投递状态和跟进日期。
5. 优化一次作品集或投递话术。

- X 上的项目方、DevRel、社区负责人
- GitHub 上文档比较弱的新项目

## 找项目的方法

优先找这些信号：

- 项目有英文文档，但中文材料很少。
- Discord / Telegram 很活跃，但公告和 FAQ 混乱。
- 测试网活动正在进行，但教程不清楚。
- 官网有产品，但缺少中文介绍或用户教程。
- 项目正在招 Ambassador、Community、Content、DevRel Intern。

## 第一条私信模板

```text
Hi，我是 cxy，正在接 Web3 兼职，方向是中文内容、项目分析、社区运营支持和轻量 dApp 前端 Demo。

我看了你们项目，感觉可以先从一个小任务帮忙，比如中文项目介绍、测试网交互教程、FAQ 整理、社区公告整理，或者钱包连接 / NFT Mint Demo。

这是我的 Web3 作品集：
https://wildcxy.github.io/web3-freelance/

我可以先试做一小版，让你们判断风格和交付质量。如果合适，再继续合作。
```

## 试做策略

不要一上来问“有没有工作”。更好的方式是主动给一个非常小的样例：

- 给项目写 1 段中文介绍。
- 把一个测试网步骤整理成 5 步教程。
- 整理 5 个常见问题 FAQ。
- 截一张页面并标出用户困惑点。
- 给他们的 Mint 流程做一个静态 UI 草稿。

试做控制在 30-60 分钟，不要免费做完整项目。

## 跟进节奏

- 第 1 天：发私信 + 作品链接。
- 第 3 天：补一次简短跟进。
- 第 7 天：发一个小试做样例。
- 第 14 天：如果没回复，停止跟进，换下一个项目。

跟进模板：

```text
Hi，我前几天发过一个 Web3 兼职协作消息。我可以先帮你们做一个很小的任务，比如中文教程、FAQ 整理或测试网交互说明。

为了方便判断，我可以先交付一个小样例，不影响你们现有流程。
```

## 投递记录表字段

建议用表格记录：

- 日期
- 项目名称
- 官网 / X / Discord
- 联系人
- 投递方向
- 是否附作品链接
- 是否试做
- 回复状态
- 下次跟进日期
- 备注

## 第一阶段目标

前 30 天不要以“马上赚很多钱”为目标，先追求：

- 发出 60-100 条有效私信。
- 做 10 个小试做样例。
- 拿到 5 次真实回复。
- 拿到 1 个小任务或长期志愿贡献机会。
- 把 1 次真实协作整理成案例文档。
